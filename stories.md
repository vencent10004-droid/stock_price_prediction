## Epic E-01: 데이터 수집

### Story 1-1: 종목 과거 주가 수집
> **As a** 개발자  
> **I want** FinanceDataReader로 종목 3년치 OHLCV 데이터를 수집하도록  
> **So that** 모델 학습에 필요한 충분한 데이터를 확보할 수 있다

**인수 조건:**
- [x] `fdr.DataReader("005930", start, end)` 정상 수집
- [x] 날짜 인덱스, 시가/고가/저가/종가/거래량 컬럼 존재
- [x] 수집된 행 수 700개 이상 확인

---

### Story 1-2: 시장 지표 수집 (KOSPI, 나스닥, 환율)
> **As a** 개발자  
> **I want** KOSPI, 나스닥, 달러/원 환율 데이터도 함께 수집하도록  
> **So that** 시장 환경 피처를 추가해 예측 정확도를 높일 수 있다

**인수 조건:**
- [x] `KS11` (KOSPI), `IXIC` (나스닥), `USD/KRW` (환율) 수집
- [x] 종목 날짜 기준으로 3개 데이터 병합 (결측치 처리)

---

### Story 1-3: 오늘 데이터 실시간 수집
> **As a** 사용자  
> **I want** 예측 실행 시 오늘 최신 데이터를 자동으로 가져오도록  
> **So that** 매번 수동으로 데이터를 업데이트하지 않아도 된다

**인수 조건:**
- [x] 예측 실행 시 최신 거래일 데이터 자동 포함
- [x] 장 마감 후(15:30 이후) 실행 시 오늘 데이터 포함 확인

---

## Epic E-02: 피처 엔지니어링

### Story 2-1: 기술적 지표 계산
> **As a** 개발자  
> **I want** RSI, MACD, 볼린저밴드, 이동평균을 자동으로 계산하도록  
> **So that** 원본 OHLCV보다 더 의미 있는 피처를 모델에 제공할 수 있다

**인수 조건:**
- [ ] RSI(14일) 계산 (0~100 범위)
- [ ] MACD, MACD Signal, MACD Histogram 계산
- [ ] 볼린저밴드 상단/하단/위치 계산
- [ ] 5일·20일·60일 이동평균 계산
- [ ] 초기 NaN 값 행 제거 후 피처 CSV 저장

---

### Story 2-2: 시장 환경 피처 추가
> **As a** 개발자  
> **I want** KOSPI, 나스닥, 환율 등락률을 피처로 추가하도록  
> **So that** 삼성전자 단독 지표만이 아닌 시장 전체 흐름을 반영할 수 있다

**인수 조건:**
- [ ] KOSPI 당일 등락률 컬럼 추가
- [ ] 나스닥 당일 등락률 컬럼 추가
- [ ] 환율 전일 대비 변화 컬럼 추가

---

### Story 2-3: 레이블 생성
> **As a** 개발자  
> **I want** 다음 날 종가 > 오늘 종가이면 1(상승), 아니면 0(하락)으로 자동 레이블링하도록  
> **So that** 지도 학습으로 모델을 학습할 수 있다

**인수 조건:**
- [ ] `label` 컬럼 자동 생성
- [ ] 마지막 행(내일 데이터 없음)은 학습에서 제외
- [ ] 상승(1)과 하락(0)의 비율 확인 (50:50에 가까운지 확인)

---

## Epic E-03: 모델 학습

### Story 3-1: 훈련/테스트 분리 및 학습
> **As a** 개발자  
> **I want** 시간 순서를 유지하면서 훈련/테스트 데이터를 분리하고 모델을 학습하도록  
> **So that** 미래 데이터가 학습에 섞이지 않는 올바른 검증이 가능하다

**인수 조건:**
- [ ] 시간 순서 유지 (앞 75% = 학습, 뒤 25% = 테스트)
- [ ] `RandomForestClassifier(n_estimators=200)` 학습
- [ ] 테스트셋 정확도 출력
- [ ] 모델 파일 저장 (`models/samsung_rf_model.pkl`)

---

### Story 3-2: 백테스트 결과 확인
> **As a** 개발자  
> **I want** 테스트셋으로 백테스트를 실행하고 상세 성능 지표를 확인하도록  
> **So that** 모델이 실제로 얼마나 유효한지 파악할 수 있다

**인수 조건:**
- [ ] 정확도(Accuracy), 정밀도(Precision), 재현율(Recall) 출력
- [ ] 날짜별 예측/실제 비교 테이블 출력
- [ ] 연속 틀린 구간 파악 (특정 시장 상황에서 약점 확인)

---

### Story 3-3: 피처 중요도 확인
> **As a** 개발자  
> **I want** 어떤 피처가 예측에 가장 큰 영향을 미치는지 확인하도록  
> **So that** 불필요한 피처를 제거하거나 중요 피처를 보강할 수 있다

**인수 조건:**
- [ ] `feature_importances_` 로 중요도 순위 출력
- [ ] 상위 10개 피처 목록 확인

---

## Epic E-04: CLI 예측 실행

### Story 4-1: 명령어 1줄로 예측 결과 출력
> **As a** 사용자  
> **I want** `python predict.py` 실행 하나로 내일 예측 결과를 바로 볼 수 있도록  
> **So that** 복잡한 조작 없이 매일 간편하게 확인할 수 있다

**인수 조건:**
- [ ] 실행 후 10초 이내 결과 출력
- [ ] 상승 ↑ / 하락 ↓ 명확히 표시
- [ ] 신뢰도(확률) % 함께 표시
- [ ] 오늘 주요 지표(RSI, MACD, KOSPI) 요약 표시

---

## Epic E-05: 웹 대시보드

### Story 5-1: 주가 차트 표시
> **As a** 사용자  
> **I want** 최근 3개월 삼성전자 주가와 이동평균선을 차트로 보도록  
> **So that** 현재 주가 위치와 추세를 한눈에 파악할 수 있다

**인수 조건:**
- [ ] 종가 + MA5 + MA20 라인 차트
- [ ] 거래량 막대 차트 (하단)
- [ ] `http://localhost:8000` 접속 시 바로 확인

---

### Story 5-2: 예측 결과 카드 표시
> **As a** 사용자  
> **I want** 대시보드 상단에 내일 예측 결과를 크고 명확한 카드로 보도록  
> **So that** 차트 분석 없이도 예측 결론을 즉시 파악할 수 있다

**인수 조건:**
- [x] 상승 예측: 빨간색 ▲ 표시 (국내 증시 관례)
- [x] 하락 예측: 파란색 ▼ 표시
- [x] 신뢰도 % + 투자의견 배지 함께 표시

---

## Epic E-06: 예측 기록 관리

### Story 6-1: 예측 결과 자동 로깅
> **As a** 사용자  
> **I want** 매일 예측 결과가 자동으로 기록되도록  
> **So that** 나중에 실제 결과와 비교해 적중률을 확인할 수 있다

**인수 조건:**
- [ ] `logs/prediction_log.csv` 에 날짜·예측·실제·정확도 기록
- [ ] 누적 적중률 계산 및 출력
