메인 대시보드

최초 진입 화면. 시가총액 상위 10개 종목을 버튼으로 제시하고, 종목 선택 시 하단 기능 패널이 나타난다.

주요 구성요소

  • 1종목 선택 그리드 — config.yaml의 10종목, 코드+종목명 표시
  • 2기능 패널 — 선택 종목명·기능 버튼·탭 (선택 전 숨김)
  • 3탭 — [예측] / [백테스트 기록] 전환
  • 4액션 버튼 — 백테스트 · 📄 PDF · 예측 조회

이벤트

  • 종목 버튼 클릭 → selectTicker() → 패널 노출 + 선택 강조
  • ‘예측 조회’ → GET /api/predict/{code} 호출 (SCR-02)
  • ‘백테스트’ → GET /api/history/{code} (SCR-03)
  • ‘📄 PDF’ → 리포트 다운로드 (SCR-04)

사용자 흐름

  • 진입 → 종목 선택 → 기능 선택(예측/백테스트/PDF)

ANNOTATION — SCR-02

예측 결과

[예측 조회] 클릭 시 호출. 조회 중 “AI 예측 중…” 로딩(30~60초) 후 결과를 렌더링한다.

주요 구성요소

  • 1강한 상승 신호 배지 — 외국인 선물+콜 동반 순매수일에만 노출(조건부)
  • 2통계 카드 6종 — 방향·신뢰도·투자의견·종가·RSI·거래량
  • 3가격대 시나리오 — 상승/하락 예상 범위·지지·저항
  • 4뉴스 감성 + 헤드라인 — 긍/중/부 막대, 관련성 정렬 10건(링크)
  • 5애널리스트 코멘트 — Gemini 생성(실패 시 규칙 기반)

이벤트

  • 헤드라인 클릭 → 새 탭으로 네이버 원문 기사 이동
  • 색상 규칙 — 상승=빨강, 하락=파랑 (국내 증시 관례)

예외 · 참고

  • 학습 모델 없는 종목 → 404 오류 안내 표시
  • 투자의견 임계값은 0.5 중심 대칭(강력매수/매수/중립/매도/강력매도)

ANNOTATION — SCR-04

자동 실행 · PDF 리포트

사용자 조작 없이 스케줄러가 매일 16:30 전체 파이프라인을 실행하고, 결과를 PDF·이메일로 전달한다. PDF는 화면에서도 즉시 다운로드 가능.

주요 구성요소

  • 1자동 파이프라인 단계 — 수집→예측·리포트→PDF→이메일
  • 2실행 로그 — 단계별 진행·결과 기록
  • 3PDF 다운로드 — GET /api/report/{code}

이벤트 · 트리거

  • 스케줄: 매일 16:30 예측 / 매주 일요일 02:00 재학습
  • ‘📄 PDF’ 클릭 → 리포트 생성·다운로드 (파일명 {code}_report.pdf)

예외 · 참고

  • 서버 구동 중에만 자동 실행 (PC·서버 상시 가동 권장)
  • 이메일 발송 실패 시 최대 2회 재시도
